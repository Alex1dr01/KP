01

Эндпоинты подключения

WS /ws/queue/{queue_id} Очередь — общий канал

Подключение к общему каналу очереди. Получает все публичные события — входы/выходы участников, изменения статусов окон, обновления позиций.

Использование

// Оператор, преподаватель, студент — все подключаются сюда
const ws = new WebSocket(`ws://217.71.129.139:4026/ws/queue/${queueId}`);
При подключении сервер сразу отправляет событие CONNECTED с текущим списком всех ожидающих участников.
WS /ws/participant/{participant_id} Участник — личный канал

Персональный канал студента. Получает личные события — вызов к окну, запросы обмена, подтверждения.

Использование

// participant_id получается из POST /participants/join
const ws = new WebSocket(`ws://217.71.129.139:4026/ws/participant/${participantId}`);
Важно: фронтенд должен держать оба соединения одновременно — очередь и личный канал.
02

События

Все сообщения имеют формат {"event": "EVENT_NAME", "data": {...}}

CONNECTED Все
Отправляется сразу при подключении. Содержит текущее состояние очереди.
{
  "event": "CONNECTED",
  "data": {
    "queue_id": "QJD-183",
    "online": 3,              // количество подключённых
    "participants": [           // текущая очередь
      {
        "id": "73036430-...",
        "nickname": "Сашка",
        "priority": "normal",
        "position": 1
      }
    ]
  }
}
USER_JOINED Все в очереди
Новый участник вошёл в очередь или был возвращён из окна.
{
  "event": "USER_JOINED",
  "data": {
    "participant_id": "73036430-...",
    "nickname": "Иван",
    "priority": "normal",    // "normal" | "low"
    "position": 3
  }
}
USER_LEFT Все в очереди
Участник покинул очередь (сам или удалён оператором).
{
  "event": "USER_LEFT",
  "data": {
    "participant_id": "73036430-...",
    "nickname": "Иван"
  }
}
POSITION_UPDATED Все в очереди
Позиции в очереди изменились (после finish или swap). Фронту нужно перезагрузить список очереди.
{
  "event": "POSITION_UPDATED",
  "data": {
    "window_number": 1,             // если после finish
    "finished_participant": "Иван", // или
    "reason": "swap"                 // если после swap
  }
}
WORKER_STATUS_CHANGED Все в очереди
Статус окна изменился — занято, свободно, занял преподаватель, пауза.
{
  "event": "WORKER_STATUS_CHANGED",
  "data": {
    "window_number": 1,
    "status": "occupied",  // "occupied"|"busy"|"idle"|"paused"
    "serving": "Иван",      // nickname, только при status=busy
    "assigned_teacher": "..." // только при назначении
  }
}
QUEUE_STATUS_CHANGED Все в очереди
Оператор изменил статус всей очереди.
{
  "event": "QUEUE_STATUS_CHANGED",
  "data": {
    "queue_id": "QJD-183",
    "status": "stopped"   // "active"|"stopped"|"closed"
  }
}
CALL_CLIENT Личное
Студента вызвали к окну. Отправляется только на личный канал участника.
{
  "event": "CALL_CLIENT",
  "data": {
    "window_number": 2,
    "message": "Вас ждут в окне №2"
  }
}
SWAP_REQUEST Личное
Другой студент предлагает обменяться местами. Нужно показать диалог подтверждения.
{
  "event": "SWAP_REQUEST",
  "data": {
    "swap_id": "uuid...",
    "from_nickname": "Иван",
    "from_position": 5,
    "to_position": 2
  }
}
SWAP_CONFIRMED / SWAP_REJECTED Личное
Результат запроса обмена местами.
// CONFIRMED — обоим участникам
{
  "event": "SWAP_CONFIRMED",
  "data": { "new_position": 3 }
}

// REJECTED — только инициатору
{
  "event": "SWAP_REJECTED",
  "data": {
    "swap_id": "uuid...",
    "message": "Swap request was rejected"
  }
}

Пример кода

JavaScript / TypeScript
const BASE_WS = 'ws://217.71.129.139:4026';

// 1. Подключение к очереди
const queueWs = new WebSocket(`${BASE_WS}/ws/queue/${queueId}`);

queueWs.onmessage = (e) => {
  const { event, data } = JSON.parse(e.data);

  switch (event) {
    case 'CONNECTED':
      setParticipants(data.participants); // начальный список
      break;

    case 'USER_JOINED':
    case 'USER_LEFT':
    case 'POSITION_UPDATED':
      reloadQueue(); // GET /participants/queue/{id}/list
      break;

    case 'WORKER_STATUS_CHANGED':
      updateWindow(data.window_number, data.status);
      break;

    case 'QUEUE_STATUS_CHANGED':
      setQueueStatus(data.status);
      break;
  }
};

// 2. Личный канал студента
const participantWs = new WebSocket(`${BASE_WS}/ws/participant/${participantId}`);

participantWs.onmessage = (e) => {
  const { event, data } = JSON.parse(e.data);

  switch (event) {
    case 'CALL_CLIENT':
      showNotification(`Вас ждут в окне №${data.window_number}`);
      break;

    case 'SWAP_REQUEST':
      showSwapDialog(data); // показать диалог
      break;

    case 'SWAP_CONFIRMED':
      setMyPosition(data.new_position);
      break;

    case 'SWAP_REJECTED':
      showMessage('Обмен отклонён');
      break;
  }
};

Переподключение

WebSocket соединение может разорваться. Рекомендуется реализовать автоматическое переподключение:

Reconnect логика
function connectWithRetry(url, onMessage, delay = 3000) {
  let ws = new WebSocket(url);

  ws.onmessage = (e) => onMessage(JSON.parse(e.data));

  ws.onclose = () => {
    setTimeout(() => connectWithRetry(url, onMessage, delay), delay);
  };

  return ws;
}
При переподключении сервер снова отправит CONNECTED с актуальным списком очереди — состояние восстановится автоматически.
